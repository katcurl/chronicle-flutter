# Chronicle 0.30.1 — исследовательская домашняя страница проекта

Chronicle 0.30.1 превращает страницу проекта из списка задач в свободное исследовательское пространство. Проект не получает обязательных стадий, типов экспериментов или лабораторного workflow: пользователь сам решает, какие вопросы, материалы и результаты относятся к нему.

## Исследовательская рамка

У каждого проекта теперь можно независимо заполнить:

- цель проекта;
- исследовательские вопросы;
- список того, что уже известно;
- список того, что ещё нужно проверить;
- закреплённые заметки с ключевыми результатами;
- связанные источники из общей библиотеки Chronicle.

Все поля необязательны. Пустой проект по-прежнему остаётся обычным свободным проектом.

## Связанные материалы

Домашняя страница автоматически собирает:

- все заметки проекта;
- все задачи проекта;
- явно связанные библиографические источники;
- вложения из Markdown-ссылок заметок в `Attachments/`;
- локальные PDF-файлы связанных источников.

Chronicle не копирует эти материалы и не переписывает заметки. Страница показывает существующие связи и позволяет держать ключевые результаты перед глазами.

## Временная шкала

Временная шкала строится без отдельного журнала и включает:

- создание проекта;
- создание и обновление заметок;
- завершение задач;
- записи времени по проекту.

Она отражает уже сохранённые события и не создаёт дополнительный обязательный процесс ведения проекта.

## Шаблоны проекта

При создании проекта можно выбрать пустой проект или один из стартовых шаблонов:

- свободное исследование;
- вычислительное исследование;
- экспериментальное исследование;
- обзор литературы.

Шаблон только заполняет исходную цель, вопросы и проверки. После создания всё можно удалить или переписать.

## Хранение и совместимость

Исследовательские поля кодируются внутри существующего поля записи проекта с версионированным префиксом. Старые обычные описания читаются без изменений. Это позволяет:

- не выполнять миграцию базы данных;
- сохранить существующую LAN-синхронизацию проектов;
- включить новые поля в резервные копии и проектный экспорт;
- не менять Markdown заметок, Vault, вложения и оформление.

Формат общей JSON-резервной копии повышен до версии 5. Декодирование старых резервных копий остаётся совместимым.
