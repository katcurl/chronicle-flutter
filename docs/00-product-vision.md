# Product vision

## One-sentence definition

Chronicle is a calm, local-first operating system for research, teaching, study, and other knowledge work, where notes, tasks, files, projects, and time records share one coherent context.

## Problem

Knowledge workers usually split one activity across several tools:

- a timer records duration;
- a task manager records intention;
- a note editor records thought;
- a file manager stores evidence and output;
- a calendar records commitments;
- a spreadsheet or report reconstructs what happened.

The split creates context loss. A completed time entry rarely explains what changed. A note rarely shows the effort invested. A task often points only vaguely to its output.

## Product promise

Every meaningful piece of work can be traced from intention to result:

`Workspace → Project → Work item → Session → Changes → Outcome`

A user should be able to answer:

- What did I do today?
- What was produced?
- Which note, file, experiment, or lesson contains the result?
- Which tasks remain blocked?
- How much time went into this project, note, or phase?
- Where did I stop, and what is the next concrete action?

## Primary users

### Researcher

Needs project history, literature notes, experiment records, files, citations, and defensible time accounting.

### Teacher or course author

Needs lecture notes, formulas, images, exercises, delivery status, revision history, and preparation time.

### Student

Needs linked notes, study projects, assignments, revision sessions, and realistic estimation of effort.

### Independent knowledge worker

Needs private project management without mandatory cloud services or subscription limits.

## Key differentiators

1. **Result-aware time tracking** — a session may point to changed notes, finished tasks, files, or a written outcome.
2. **Open storage** — Markdown and attachments remain usable outside Chronicle.
3. **Context continuity** — the application remembers where work stopped and exposes the next action.
4. **Project history** — an event timeline reconstructs how an idea or deliverable developed.
5. **Calm interface** — information density is progressive rather than constantly maximal.
6. **Offline by default** — core functionality never depends on an account or server.

## MVP scope

The first dependable release includes:

- workspaces and projects;
- tasks and work items;
- timer and manual time records;
- Markdown notes with LaTeX and images;
- links between notes, tasks, projects, and sessions;
- local database and ordinary files;
- daily view and basic reports;
- import, export, and backups;
- Android first, with architecture suitable for desktop later.

## Explicit non-goals before v1.0

- real-time multi-user editing;
- enterprise billing and payroll;
- a general-purpose cloud drive;
- AI that silently changes user content;
- cloning the visual identity of Toggl, Obsidian, Notion, or another product;
- plugin compatibility with the full Obsidian ecosystem.
