# Chronicle v0.16 — Sync Core

Этот этап добавляет локальное ядро двусторонней синхронизации без изменения QR-сопряжения и LAN-транспорта.

## Что реализовано

- сериализация пакетов журнала изменений;
- чтение журнала после заданной локальной последовательности;
- продвижение курсора даже через события, пришедшие от того же peer;
- защита от обратного эха изменений;
- идемпотентное применение по `changeId`;
- детерминированное разрешение конфликтов по `revision`, `changedAt`, `changeId`;
- применение проектов, задач, заметок, версий заметок и записей времени;
- tombstones для удалённых задач и заметок;
- атомарное применение пакета в Drift/SQLite;
- перезагрузка `AppStore` после входящих изменений;
- тесты JSON, идемпотентности, конфликтов, удалений и echo prevention.

## Следующий этап

LAN-транспорт будет передавать `SyncJournalBatch` между уже доверенными устройствами, обновлять `SyncCursor` и показывать результат обмена в интерфейсе.
