# Chronicle v0.7 — Data Core

## Цель

Перевести приложение с одной JSON-строки в SharedPreferences на SQLite, не меняя привычный мобильный интерфейс и не теряя данные установленной версии.

## Реализовано

- SQLite-база `chronicle.db`;
- таблицы проектов, задач, заметок, временных записей и состояния приложения;
- индексы для основных выборок;
- one-time импорт данных из ключа `chronicle_data_v5`;
- seed-данные только при первом запуске;
- сохранение активного таймера между перезапусками приложения;
- soft delete заметок;
- JSON backup/import;
- абстракция `AppRepository`;
- production-репозиторий SQLite;
- in-memory репозиторий для тестов;
- тесты сериализации и сохранения данных.

## Источник истины

Начиная с v0.7, структурированные данные хранятся в SQLite. Markdown Vault будет добавлен отдельным слоем в следующих версиях. Текст заметок пока хранится в SQLite, чтобы сначала стабилизировать миграции и CRUD.

## Миграция с v0.6

При первом запуске v0.7 приложение:

1. создаёт SQLite-базу;
2. проверяет старый JSON в SharedPreferences;
3. импортирует найденные проекты, задачи, заметки и сессии;
4. помечает базу инициализированной;
5. далее использует только SQLite.

Старый JSON не удаляется автоматически, чтобы оставалась аварийная копия на период тестирования v0.7.
